## 📌 Project Overview

Cardiovascular disease is the leading cause of death worldwide. This project builds a
clinically-informed, end-to-end machine learning pipeline to predict heart disease risk
from patient data, and uses **SHAP (Shapley Additive Explanations)** to make every
prediction fully transparent and interpretable.

Three models were trained and evaluated using Stratified 5-Fold Cross Validation.
**Logistic Regression outperformed both Random Forest and XGBoost on all metrics** —
a result that makes clinical sense given the small, clean, structured nature of this dataset.
The final model was selected purely based on CV results, not assumptions.

Developed as part of **MSc Computer Science — Emerging AI** and expanded into a
production-grade portfolio project.

## ✨ Key Features

- **Proper ML pipeline** — raw data → EDA → preprocessing → modelling → explainability → app
- **Three models compared** — Logistic Regression (champion) · Random Forest · XGBoost
- **Model selection driven by data** — Logistic Regression won on all 5 CV metrics
- **Stratified 5-Fold Cross Validation** for reliable, unbiased evaluation
- **Outlier removal** using IQR method on continuous numerical features
- **Feature Engineering** — 5 new domain-informed clinical features
- **SMOTE** applied on training data only to handle class imbalance
- **StandardScaler** fitted on training data, applied to test data (no data leakage)
- **Full SHAP suite** — Summary · Bar · Waterfall · Force · Dependence plots
- **SHAP LinearExplainer** — correct explainer type for linear models
- **Streamlit web app** — live patient input, real-time prediction, per-patient SHAP explanation

## 📊 Dataset

**Cleveland Heart Disease Dataset** — UCI Machine Learning Repository
- 297 patients · 13 clinical features · Binary target (0=No Disease, 1=Disease)
- No missing values · Mild class imbalance handled with SMOTE

**Source:** [Kaggle — Cleveland Heart Disease UCI](https://www.kaggle.com/datasets/cherngs/heart-disease-cleveland-uci)

### Feature Reference

| Feature | Description |
|---|---|
| age | Age in years |
| sex | 1 = Male, 0 = Female |
| cp | Chest pain type (0=Typical Angina, 1=Atypical, 2=Non-anginal, 3=Asymptomatic) |
| trestbps | Resting blood pressure (mmHg) |
| chol | Serum cholesterol (mg/dl) |
| fbs | Fasting blood sugar > 120 mg/dl (1=True, 0=False) |
| restecg | Resting ECG (0=Normal, 1=ST-T Abnormality, 2=LV Hypertrophy) |
| thalach | Maximum heart rate achieved |
| exang | Exercise induced angina (1=Yes, 0=No) |
| oldpeak | ST depression induced by exercise |
| slope | Slope of peak exercise ST segment (0=Upsloping, 1=Flat, 2=Downsloping) |
| ca | Number of major vessels coloured by fluoroscopy (0-3) |
| thal | Thalassemia (0=Normal, 1=Fixed Defect, 2=Reversible Defect) |
| condition | TARGET — 0=No Heart Disease, 1=Heart Disease |

## 🧠 Model Results — Stratified 5-Fold Cross Validation

| Model | Accuracy | Precision | Recall | F1 | ROC-AUC |
|---|---|---|---|---|---|
| **Logistic Regression ★** | **0.8248** | **0.8466** | **0.8053** | **0.8222** | **0.9047** |
| Random Forest | 0.8047 | 0.8064 | 0.8053 | 0.8045 | 0.8920 |
| XGBoost | 0.8088 | 0.8194 | 0.7973 | 0.8065 | 0.8985 |

**Why Logistic Regression won:**
This is a well-known result in ML — simpler models often outperform complex ensemble methods
on small, clean, structured datasets. With only ~278 rows after cleaning, XGBoost and
Random Forest do not have enough data to leverage their complexity advantage.
Logistic Regression's linear decision boundary is well-suited to the relatively linear
feature relationships in this clinical dataset.

## 🔍 SHAP Explainability

SHAP (Shapley Additive Explanations) provides both global and individual explanations.
`shap.LinearExplainer` was used — the correct explainer for Logistic Regression.

| Plot | What it shows |
|---|---|
| Summary (Beeswarm) | Which features matter most across all patients |
| Bar Chart | Mean absolute SHAP values per feature |
| Waterfall | Why the model made a specific prediction for one patient |
| Force Plot | Feature tug-of-war toward vs away from disease |
| Dependence | How thalach interacts with ca across all patients |
